[**Goals**]

- G-1: Per-instruction DUT/REF state comparison; halt on first divergence with a structured `DiffMismatch` report.
- G-2: Compare PC + GPRs + privilege mode + a whitelisted CSR set (masked) — whitelist auto-derived from `csr_table!` `@ difftest` annotations.
- G-3: Support QEMU as a reference via the GDB Remote Serial Protocol (no patches to QEMU required).
- G-4: Support Spike as a reference via a thin C++ FFI wrapper, statically linking libriscv / libsoftfloat / libfesvr / libdisasm.
- G-5: Stay feature-gated — `cargo build --features difftest` enables; default build pays zero runtime cost.

[**Non-goals**]

- NG-1: No automated CI integration — running difftest in CI requires QEMU and Spike in the runner image; deferred.
- NG-2: No bidirectional sync — DUT is the leader; REF follows. Divergences flow DUT → user, not back into REF.
- NG-3: No floating-point register comparison — FP state and `fcsr` are not yet on the whitelist.

[**Architecture**]

```
xemu/xdb/src/difftest/
├── mod.rs          DiffBackend trait + DiffHarness + DiffMismatch + diff_contexts() free fn
├── gdb.rs          GdbClient — GDB-RSP framing, register packing for QEMU
├── qemu.rs         QemuBackend — wraps GdbClient + initial-state sync + step loop
└── spike.rs        SpikeBackend — extern "C" FFI to spike_init / step / get_pc / get_gpr / get_csr / ...

xemu/xcore/src/arch/riscv/cpu/csr.rs
└── DIFFTEST_CSRS: &[(CsrAddr, u64)]   // auto-generated from csr_table! `@ difftest` rows

tools/difftest/spike/                  C++ wrapper exposing extern "C" functions over libriscv/libfesvr
```

QEMU is driven via GDB-RSP with `monitor sstep 0x7` (NOIRQ + NOTIMER) + `PhyMemMode:1`. Spike is driven directly via its in-process C++ API wrapped in `extern "C"` shims. Both backends produce a `CoreContext` snapshot that `diff_contexts` compares against the DUT's.

[**Data Structure**]

```rust
pub trait DiffBackend {
    fn step          (&mut self)                                    -> Result<(), String>;
    fn read_context  (&mut self)                                    -> Result<CoreContext, String>;
    fn sync_state    (&mut self, ctx: &CoreContext)                 -> Result<(), String>;
    fn write_mem     (&mut self, addr: usize, data: &[u8])          -> Result<(), String>;
    fn name          (&self)                                        -> &str;
}

pub struct DiffMismatch {
    pub inst_count: u64,
    pub field:      &'static str,    // "pc" / "x5" / "mstatus" / ...
    pub dut:        u64,
    pub ref_:       u64,
    pub mask:       u64,
}

pub struct DiffHarness { /* backend + inst counter */ }
pub struct QemuBackend { /* GdbClient + state */ }
pub struct SpikeBackend { /* *mut SpikeCtx FFI handle */ }
pub struct GdbClient { /* TCP socket + RSP framing */ }
```

impl DiffHarness {

[**Constraints**]

- C-1: Difftest is feature-gated by `#[cfg(feature = "difftest")]`; default builds compile out the module entirely — `xemu/xdb/Cargo.toml`.
- C-2: The CSR whitelist is generated by `csr_table!`'s `@ difftest` annotation; manual edits to `DIFFTEST_CSRS` are forbidden — `xemu/xcore/src/arch/riscv/cpu/csr.rs:120`.
- C-3: QEMU is configured with `sstep = 0x7` (NOIRQ | NOTIMER | NOTC) and `PhyMemMode:1` so single-step matches DUT's deterministic execution — `xemu/xdb/src/difftest/qemu.rs`.
- C-4: Spike is statically linked via a C++ wrapper at `tools/difftest/spike/`; the wrapper exposes `extern "C"` shims so xdb has no C++ dependency.
- C-5: MMIO reads / writes (regions outside RAM) skip per-step comparison; values from the DUT are pushed into REF via `write_mem` to keep memory state in sync — `xemu/xdb/src/difftest/mod.rs`.
- C-6: A `DiffMismatch` halts the harness immediately; the user sees one structured report, not a continuing diff stream.
- C-7: `DiffBackend::name()` returns `"qemu"` or `"spike"`; the xdb `dt attach <name>` command dispatches on this string.
