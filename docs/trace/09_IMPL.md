## Completed Scope

Phase 5: Debugging & Observability for xemu.

### xcore Debug Infrastructure

- **`DebugOps` trait** — unified arch-agnostic facade for breakpoint management + read-only inspection (registers, memory, disassembly). Always compiled, zero overhead.
- **`Breakpoint` struct** — stable user-visible IDs (`u32`, monotonic, never reused).
- **Breakpoint check in `step()`** — cfg-gated (`cfg(feature = "debug")`): only the bp check and per-instruction `trace!()` log are behind the feature flag. Everything else compiles unconditionally.
- **`format_mnemonic()`** — converts `DecodedInst` to human-readable string for all R/I/S/B/U/J/C instruction formats.
- **`RVReg::name()` / `RVReg::from_name()`** — GPR ABI names + `x0`–`x31` + `fp` alias.
- **`CsrAddr::from_name()`** — generated by extended `csr_table!` macro.
- **`XError::DebugBreak(usize)`** — non-fatal debugger stop variant.
- **CPU pass-through** — `CPU<Core: DebugOps>` exposes `add_breakpoint`, `remove_breakpoint`, `list_breakpoints`, `set_skip_bp`, `debug_ops()`, `pc()`, `is_terminated()`.

### xdb Debugger Commands

- **Breakpoints**: `b <addr>`, `bd <id>`, `bl`
- **Watchpoints**: `w <expr>`, `wd <id>`, `wl` — expression-based, validated at creation, evaluated per-step in xdb
- **Examine**: `x/Ni [addr]` (disassemble), `x/Nx [addr]` (hex), `x/Nb [addr]` (bytes) — GDB-style pre-parser via `Regex::new` + `OnceLock`
- **Print**: `p <expr>` — evaluate and print expression
- **Info**: `info reg [name]` — register dump (all or one)
- **Expression evaluator** — recursive descent parser: `$reg`, `0xHEX`, decimal, `+-*/%`, `==`, `!=`, `*deref`, `()`

### Execution Logging

Replaced trace ring buffers with proper `log!()` at different levels:

| Level | Events |
|-------|--------|
| `trace` | Per-instruction execution (pc + raw + mnemonic), memory load/store (addr + size + value), UART THR output |
| `debug` | Trap handler (cause + target mode), mret/sret (return pc + privilege), CSR side effects (satp, pmpcfg, pmpaddr), PLIC claim/complete/evaluate, ACLINT writes, Bus reset |
| `info` | CPU reset, program exit, file load, MMIO device registration |

### Build System

- `debug` feature on xcore: gates bp check in `step()` and per-instruction `trace!()` logging
- `debug` feature on xdb: forwards to `xcore/debug`
- Makefile: `BATCH=n` (interactive) → enables debug; `BATCH=y` (batch/CI) → disables debug
- All debug commands always compiled in xdb — only the critical-path hooks are feature-gated (2 cfg blocks total)

## Deviations from Plan

| Deviation | Reason |
|-----------|--------|
| No `chumsky` crate | Custom recursive descent is cleaner and dependency-free (~200 lines) |
| No trace ring buffers | Master directive M-001(06): use xlogger log levels instead |
| All addresses physical | Bare-metal identity mapping. Documented in command help. |
| `DebugOps` always compiled | Master directive: reduce cfg gates. Only bp check + trace!() per-inst cfg-gated |
